# 專案開發計劃與記錄

這個資料夾存放專案的開發計劃、執行記錄與技術決策文件，旨在：

1. **記錄開發歷程**：完整保留每個功能的需求分析、實作計劃和執行過程
2. **提供上下文資訊**：協助未來的 AI 助手或開發者快速理解專案演進
3. **技術決策文件**：記錄重要的技術選擇和架構決策

## 📁 文件結構

計劃文件採用編號命名，按時間順序排列：

```
planning/
├── README.md                           # 本說明文件
├── 000-initial-setup.md                # 專案初始化與基礎架構
├── 001-node-upgrade.md                 # Node.js 24 升級計劃
├── 002-homepage-article-list.md        # 首頁文章列表實作
├── 003-issue-comments-integration.md   # Issue 留言整合功能
├── 004-comments-newline-fix.md         # 留言換行格式修復
├── 005-article-list-enhancement.md     # 文章列表功能增強
└── archive/                            # 已棄用或過時的計劃
```

## 📝 文件命名規範

- **格式**：`{編號}-{簡短描述}.md`
- **編號**：從 000 開始的三位數字，按時間順序遞增
- **描述**：使用小寫英文，單詞間以 `-` 連接（kebab-case）
- **範例**：`003-issue-comments-integration.md`

## 📋 計劃文件應包含的內容

每個計劃文件應包含以下章節：

1. **問題陳述**：要解決的問題或要實現的功能
2. **需求分析**：具體的需求清單和驗收標準
3. **技術方案**：選擇的技術方案和實作方式
4. **任務拆解**：將大任務拆分為可執行的小任務
5. **實作記錄**：執行過程中的關鍵決策和遇到的問題
6. **測試驗證**：測試方法和驗證結果
7. **完成狀態**：✅ 已完成、🚧 進行中、⏸️ 暫停、❌ 已取消

## 🎯 使用方式

### 開始新功能時

1. 建立新的計劃文件（使用下一個編號）
2. 填寫問題陳述和需求分析
3. 設計技術方案並拆解任務
4. 在實作過程中持續更新執行記錄

### 查找歷史資訊時

1. 查看 README.md 的文件索引
2. 根據功能關鍵字找到相關計劃文件
3. 閱讀完整的需求、實作和決策記錄

### 提供給 AI 助手時

1. 將相關計劃文件的路徑提供給 AI
2. AI 可以讀取文件了解專案背景和技術決策
3. 避免重複詢問已解決的問題

## 📌 注意事項

1. **保持更新**：計劃文件應在功能開發過程中持續更新，而非事後補充
2. **記錄決策**：重要的技術決策和取捨應詳細記錄原因
3. **避免重複**：使用連結引用其他文件，避免內容重複
4. **清晰簡潔**：使用清楚的標題和列表，方便快速掃描
5. **版本控制**：所有計劃文件都納入 Git 版本控制

## 🤖 AI 助手使用指南

如果您是 AI 助手，閱讀這些計劃文件可以幫助您：

- **了解專案架構**：技術棧、檔案結構、核心流程
- **理解設計決策**：為什麼選擇這種實作方式
- **避免重複工作**：已實現的功能和已解決的問題
- **保持一致性**：遵循既有的程式碼風格和設計模式
- **快速上手**：減少詢問背景問題的次數

建議閱讀順序：
1. 先讀 `000-initial-setup.md` 了解整體架構
2. 根據當前任務找相關功能的計劃文件
3. 查看最近的計劃文件了解最新狀態
